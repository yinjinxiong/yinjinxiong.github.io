搭建hexo博客之个性化设置

发表于 2018-05-14 | 分类于 hexo | 阅读次数:

个性化设置主要是对配置文件的修改，现在假设默认博客的初始化目录为${blog}，hexo分为两类配置文件，一种是站点配置文件【在${blog}/_config.yml】，一种是主题配置文件【在${blog}/themes/${themeName}/_config.yml】

配置站点信息

在 站点配置文件 找到”Site”,说明如下图：

设置主题

  • 安装你需要的主题，以next为例:
    在${blog}目录下载主题 
    git clone https://github.com/iissnan/hexo-theme-next themes/next
  • 使用主题
    在站点配置文件找到 “theme”，然后写上你想要的主题 
    # Extensions        
## Plugins: https://hexo.io/plugins/        
## Themes: https://hexo.io/themes/        
theme: next
  • 验证主题 
    hexo s --debug
    在浏览器中查看 http://localhost:4000/，验证主题是否正确配置

创建本地搜索引擎

  • 安装搜索引擎插件 
npm install --save hexo-generator-search
  • 在站点配置文件 增加以下配置： 
    search:
  path: search.xml
  field: post
    此配置表示为全站的post类型文章建立索引以便搜索，如果需要把所有的文章都纳入索引，可以把 field改成 all

增加阅读数量

采用leancloud第三方辅助我们进行文章统计，next已经支持leancloud，所以只需要去leancloud网站注册，创建新应用，名为“个人博客”
在菜单 存储->数据->创建Class 名字为Counter->设置数据条目的默认为无限制
在菜单 设置->应用Key里面存有App ID和App Key获取出来保存，并在主题配置文件里面进行配置： 

leancloud_visitors:
  enable: true
  class: Counter
  app_id: 填写APPID
  app_key: 填写APPKEY

设置博客菜单以及菜单图标

在主题配置文件增加以下配置： 

menu:
  home: / || home #主页
  about: /about/ || user #关于页
  tags: /tags/ || tags #标签页
  categories: /categories/ || th #类目页
  archives: /archives/ || archive # 文章页面
  #schedule: /schedule/ || calendar
  #sitemap: /sitemap.xml || sitemap
  #commonweal: /404/ || heartbeat

# Enable/Disable menu icons. 以下是菜单对应的图标
menu_icons:
  enable: true
  home: home
  about: user
  categories: th
  tags: tags
  archive: archive
